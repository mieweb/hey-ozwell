# prod/js — Hey Ozwell Browser Runtime

Browser wake-word detection demo + library, built on the [Hey Buddy](https://github.com/painebenjamin/hey-buddy) ONNX pipeline.

## Pipeline (in-browser)

1. `AudioWorklet` captures mic at native rate and resamples to 16 kHz.
2. **Mel-spectrogram** ONNX (loaded from `huggingface.co/benjamin-paine/hey-buddy`) → 32 mel bins per frame.
3. **Speech-embedding** ONNX (same HF repo) → 96-dim embedding, one per 80 ms frame.
4. **Wake-word MLP** ONNX (one per phrase, ours from [`../../model/`](../../model/)) → score in `[0, 1]` over a 16-frame window.
5. **Silero VAD** ONNX gates activations to suppress non-speech triggers.

Default ships two Ozwell wake words: `hey-ozwell` and `ozwell-i'm-done`.

## Run the demo locally

```bash
cd prod/js
npm install
npm run worklet   # bundles the AudioWorklet
npm run build     # bundles the page script
PORT=3001 npm start
# open http://localhost:3001
```

Click **Initialize** in the page to load models, then say "Hey Ozwell" — the UI shows live scores and a chime on detection.

## Layout

| Path | Purpose |
|---|---|
| `index.html` | Demo page |
| `src/` | Page sources (`index.js`, `hey-buddy.js` runtime, `audio.js`, `models.js`, `onnx.js`, `mutex.js`, `helpers.js`, styles, AudioWorklet `models/`) |
| `worklet.js`, `worklet.config.js` | AudioWorklet entry + webpack config |
| `webpack.config.js`, `babel.config.json` | Bundler config |
| `server.js` | Tiny Express dev server (port via `PORT`, default 3001) |
| `models/` | Trained wake-word ONNX models (LFS) — generated by [`../../model/tools/convert.py`](../../model/tools/convert.py) |
| `dist/` | Webpack output (gitignored) |

## Adding a new wake word

1. Train it in [`../../model/`](../../model/) and convert with `python tools/convert.py --input ... --output ../prod/js/models/<name>.onnx`.
2. Register the model in `src/index.js` (look for the existing wake-word list).
3. Rebuild with `npm run build`.
